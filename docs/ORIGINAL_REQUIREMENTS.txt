Build a complete downloadable Windows chess-teacher application.

## 1. Main goal

Create an installable chess training application similar in purpose to Chess.com analysis and lessons.

The application must allow users to:

* Play chess against the computer
* Play both sides on a free analysis board
* Analyze completed games
* Receive move evaluations
* See best-move arrows
* Get explanations for mistakes
* Practice tactical puzzles
* Study openings
* Track progress
* Save games locally
* Import and export PGN files

The program must be a real desktop application, not a website.

## 2. Required technology

Use:

* C#
* .NET 8 or newer stable version
* Avalonia UI
* Stockfish as the chess engine
* UCI protocol for communication with Stockfish
* SQLite for saved data
* JSON for lessons and settings
* MVVM architecture
* Windows x64 as the first supported platform

Do not use:

* Python
* Flask
* Django
* Electron
* Browser-based HTML as the main application
* A local web server
* Node.js as a runtime dependency
* Unity unless a future 3D mode is added

The final user must not need to install Python, Node.js or additional developer software.

## 3. Stability requirements

The application must:

* Never automatically open or run after generation
* Never automatically launch a browser
* Never automatically start Stockfish during installation
* Never freeze the interface while Stockfish is calculating
* Run all engine calculations asynchronously
* Allow engine analysis to be cancelled
* Stop the Stockfish process safely when the application closes
* Prevent multiple unnecessary Stockfish processes
* Recover gracefully if Stockfish crashes
* Show clear error messages
* Avoid excessive RAM and CPU use
* Limit engine threads and hash memory through settings
* Save data safely without corrupting the database

Do not preview, execute or auto-open the generated application during development delivery.

Only create the complete project files and build instructions.

## 12. Game analysis

After a game, analyze every move.

For each move calculate:

* Evaluation before the move
* Evaluation after the move
* Best engine move
* Played move
* Centipawn loss
* Best continuation
* Tactical consequences
* Whether mate was missed
* Whether a forced mate was allowed

Classify moves as:

* Brilliant
* Great
* Best
* Excellent
* Good
* Book
* Inaccuracy
* Mistake
* Blunder
* Missed win
* Forced move

Do not copy Chess.com’s private algorithm or branding.

Create an original move-classification system.

## 13. Move-classification logic

Use centipawn loss as one factor.

Suggested initial thresholds:

* Best: loss below 0.15 pawns
* Excellent: loss below 0.35 pawns
* Good: loss below 0.75 pawns
* Inaccuracy: loss below 1.50 pawns
* Mistake: loss below 3.00 pawns
* Blunder: loss of 3.00 pawns or more

Improve the classification using:

* Mate score changes
* Change from winning to equal
* Change from equal to losing
* Material loss
* Missed tactical opportunities
* Only legal or forced move
* Sacrifice detection
* Difficulty of finding the move

Thresholds must be configurable.

## 14. Average centipawn loss

Calculate average centipawn loss separately for White and Black.

Use:

Average centipawn loss =
Total centipawn loss divided by number of analyzed moves

Ignore forced moves when appropriate.

Cap extreme mate-score conversions to prevent unrealistic averages.

## 50. Definition of completion

The project is complete only when a normal Windows user can:

1. Download ChessTeacherSetup.exe
2. Install it
3. Open the application
4. Start a game
5. Move pieces normally
6. Play against Stockfish
7. Finish or resign the game
8. Analyze the game
9. See move evaluations
10. Read explanations
11. Save the game
12. Close and reopen the program
13. Find the saved game and progress

The application must achieve this without requiring Python, Node.js, command-line tools or programming knowledge.
